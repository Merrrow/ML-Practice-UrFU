## Задание 1: Создание и манипуляции с тензорами

### 1.1 Создание тензоров
- `torch.rand(3, 4)` — создаёт тензор размером 3×4 со случайными числами от 0 до 1.
- `torch.zeros(2, 3, 4, dtype=torch.int)` — создаёт тензор 2×3×4, заполненный нулями целого типа.
- `torch.ones(5, 5, dtype=torch.int)` — создаёт тензор 5×5, заполненный единицами.
- `torch.arange(16).reshape(4, 4)` — создаёт одномерный тензор от 0 до 15 и преобразует его в форму 4×4.

### 1.2 Операции с тензорами
- `.T` — транспонирование тензора.
- `torch.matmul(A, B)` или `A @ B` — матричное умножение.
- `A * B.T` — поэлементное умножение (оператор `*`).
- `.sum()` — вычисление суммы всех элементов тензора.

### 1.3 Индексация и срезы
- `t[0]` — извлечение первой строки.
- `t[:, :, -1]` — последний столбец (последний индекс по третьей оси).
- `t[2, 1:3, 1:3]` — подматрица 2×2 из центра (индексы 2 по первой оси, срезы по двум другим).
- `t[::2, ::2, ::2]` — выборка элементов с чётными индексами по всем осям (шаг 2).

### 1.4 Работа с формами
- `.reshape(2, 12)`, `.reshape(3, 8)`, `.reshape(4, 6)`, `.reshape(2, 3, 4)`, `.reshape(2, 2, 2, 3)` — изменение формы тензора из 24 элементов в указанные размерности.

## Задание 2: Автоматическое дифференцирование

### 2.1 Простые вычисления с градиентами
- `torch.tensor(..., requires_grad=True)` — создание тензоров, для которых будут вычисляться градиенты.
- Вычисление `f = x**2 + y**2 + z**2 + 2*x*y*z` — составление функции.
- `.backward()` — вычисление градиентов функции по всем переменным, имеющим `requires_grad`.
- `.grad` — доступ к вычисленным градиентам.

### 2.2 Градиент функции потерь (MSE)
- Функция `MSE(y_pred, y_true)` — реализована как `((y_pred - y_true)**2).sum() / n`, где `n = y_true.numel()`.
- Построение линейной модели `y_pred = w * x + b` с `requires_grad=True` для `w` и `b`.
- `loss.backward()` — вычисление градиентов `loss` по `w` и `b`.
- `.grad` — получение значений градиентов.

### 2.3 Цепное правило
- `torch.sin(x**2 + 1)` — составная функция.
- `.backward()` — автоматическое вычисление производной.
- `torch.autograd.grad(f, x)[0]` — альтернативный способ явного вычисления градиента (использован для проверки).
- `torch.allclose()` — сравнение результатов (в коде добавлены assert для проверки).

## Задание 3: Сравнение производительности CPU vs CUDA

### 3.1 Подготовка данных
- `torch.rand(64, 1024, 1024)`, `torch.rand(128, 512, 512)`, `torch.rand(256, 256, 256)` — создание трёх тензоров разных размеров для тестирования.

### 3.2 Функция измерения времени
- `time.time()` — измерение времени на CPU.
- `torch.cuda.Event(enable_timing=True)` и методы `.record()`, `.elapsed_time()` — точный замер времени на GPU.
- `torch.cuda.synchronize()` — синхронизация потоков GPU для корректного замера.
- Прогрев (несколько запусков перед замером) для исключения начальных задержек.

### 3.3 Сравнение операций
- Операции: `@` (матричное умножение), `+` (поэлементное сложение), `*` (поэлементное умножение), `.T` (транспонирование), `torch.sum()`.
- Для каждого размера матрицы измеряется время выполнения на CPU и GPU, вычисляется ускорение (`cpu_time / gpu_time`).

### 3.4 Анализ результатов
- Вывод сделан в виде комментариев/текста, отвечающих на вопросы:
  - Наибольшее ускорение получают матричное умножение и сумма (высокая вычислительная плотность).
  - Транспонирование может давать меньшее ускорение из-за ограничений пропускной способности памяти.
  - С ростом размера ускорение растёт, так как GPU эффективнее загружается.
  - Передача данных между CPU и GPU через PCIe — узкое место, поэтому минимизируем перемещения.
